# Классификатор отзывов с использованием LLM (OpenRouter)

Автоматическое определение тональности и темы текстовых отзывов с помощью языковой модели через API OpenRouter.

## Возможности

- Работа с входными данными, в формате CSV
- Классификация тональности: `positive`, `negative`, `neutral`, `mixed`, `unknown`
- Определение темы: `delivery`, `product_quality`, `fit`, `seller`, `price`, `other`, `unknown`
- Использование JSON-схемы (`response_format`), для строгого формата ответа
- Надёжный парсинг - всегда возвращается предсказуемый словарь

## Требования

- Python 3.8+
- Аккаунт [OpenRouter](https://openrouter.ai/) и API-ключ

## Установка

1. Сохраните скрипт `script.py`.
2. Установите необходимые библиотеки:
```bash
pip install pandas requests python-dotenv
```
3. Создайте файл `.env` в той же директории и добавьте в него свой API-ключ:
```ini
OPENROUTER_API_KEY=ваш_ключ
```

## Подготовка данных

Входной CSV-файл (по умолчанию reviews.csv) должен:
- использовать разделитель ;
- содержать обязательный столбец review (с текстом отзывов)
- быть сохранён в кодировке UTF-8

## Запуск

```bash
python script.py
```
После выполнения будет создан файл reviews_classified.csv с добавленными колонками sentiment и topic, в терминале, при успешном выполнении скрипта появится следующее уведомление - `Saved: reviews_classified.csv`.

## Настройки

В начале скрипта вы можете изменить параметры конфигурации:
- Названия входного и выходного файлов:
```ini 
INPUT_CSV = "ваш_файл"
OUTPUT_CSV = "ваш_файл"
```
- В скрипте используется бесплатная модель: 
```ini 
MODEL = "openai/gpt-oss-120b:free"
``` 
При утрате актуальности этой модели, можно использовать другую бесплатную модель с OpenRouter или обратиться к доступным платным вариантам. 

## Принцип работы

- Чтение CSV-файла построчно
- Отправка каждого отзыва в OpenRouter API вместе с JSON-схемой
- Получение и парсинг JSON вида {"sentiment": "...", "topic": "..."}
- Добавление результатов в DataFrame и сохранение в новый CSV

## Начальные данные и результат

В репозитории можно обратиться, к исходному файлу "reviews.csv", в котором содержится датасет(размерностью в 50 строк) с двумя колонками: id и review. В колонке review хранятся текстовые отзывы на русском языке, а в id - уникальный идентификатор для каждого отзыва. Также можно обраться к выходному файлу "reviews_classified.csv", в котором можно увидеть результат работы скрипта - датасет с двумя изначальными колонками и двумя новыми - sentiment(тональность) и  topic(тема).

## Минусы

Автоматическая классификация на данной выборке показывает хорошие результаты только для простых, однозначных отзывов. Проблема заключается в использовании модели с огрнаниченным функционалом, замена модели на более качественый аналог позволит улучшить итоговый результат.
